# Renesance v2.5 -- Восстановление и Экспорт Ассетов TTS

Ультимативный инструмент для спасения модов, починки сломанных Cloud-ссылок и полного экспорта карт Tabletop Simulator.
Программа находит файлы в вашем локальном кэше игры и помогает сохранить их или интегрировать обратно в сейв.

---

## 🚀 Быстрый старт

Программа полностью портабельна и работает "из коробки" для 90% пользователей Windows!
Просто скачайте папку, запустите **`START.bat`** (создастся красивое меню в консоли).

При первом запуске автоматически появится файл `renesance_config.txt`. 
В нём используются стандартные переменные Windows (по типу `%USERPROFILE%\Documents...`), поэтому в большинстве случаев **вам не придется настраивать пути вручную!**

Но если у вас игра установлена на другом диске (или вы играете с пиратки), откройте "Настройки" (Пункт 1, затем Пункт 5) и добавьте ваш путь к папке игры.

---

## 🛠️ Главные функции и как ими пользоваться

В программе есть три основных сценария использования:

### Сценарий 1: Автоматическая починка битого сейва (Авто-патч)
*У вас есть сейв с кучей битых облачных ссылок (DropBox, Pastebin и т.д.), но у вас на ПК эти файлы точно есть (вы заходили на эту карту год назад и всё закэшировалось).*

1. В главном меню включите **Режим отладки (Пункт 8)**. Это временно скроет ваш основной кэш от игры, чтобы мы могли точно увидеть, какие ссылки мертвы.
2. Запустите Tabletop Simulator и **загрузите битый сейв**.
3. Дождитесь конца загрузки. Игра начнет жаловаться на ошибки красным текстом в чате (ошибка загрузки URL). Это нормально!
4. **Закройте игру**.
5. В программе выберите **Пункт 9 (Найти ошибки и пропатчить сейв)**.
   * Программа прочитает системный лог игры (`Player.log`), найдет каждую ссылку, на которую ругалась игра.
   * Программа найдет копии этих файлов в вашем "скрытом" кэше и свяжет одно с другим.
   * В результате рядом с вашим сейвом появится новый файл: `ВашСейв_PATCHED.json`.
6. Отключите Режим отладки (повторно нажав Пункт 8).
7. Зайдите в игру, загрузите `ВашСейв_PATCHED.json`. Все мертвые облачные ссылки заменены на локальные файлы `file:///...`!
8. Откройте `Cloud Manager` в Tabletop Simulator и нажмите **Upload All**, чтобы залить умершие файлы на ваше собственное облако Steam!

### Сценарий 2: ПОЛНЫЙ ЭКСПОРТ всех ассетов
*Вы хотите собрать ВСЕ модели, картинки и звуки с карты в одну отдельную папку (например, чтобы заархивировать, скинуть другу, или залить на личный сервер).*

1. Запустите программу, выберите **Пункт 5 (Полный экспорт)**.
2. Выберите нужный сейв.
3. Программа прочитает сейв, найдет все уникальные ссылки, просканирует ВЕСЬ ваш настроенный кэш (см. секцию Проблемы) и скопирует все существующие файлы в папку `Exported_Mods` внутри папки программы.
4. Вы получите папку со всеми нужными ресурсами рассортированными по `Images`, `Models` и т.д.

### Сценарий 3: Частичный экспорт через TTS Куб
*Если вам нужно извлечь файлы только одного конкретного объекта на столе.*

1. Выберите **Пункт 2 (Собрать TTS куб)**. Это создаст инструмент.
2. В самой игре зайдите в `Objects -> Saved Objects -> "Renesance"` и вытащите оранжевый куб на стол.
3. Положите нужные объекты возле куба. Нажмите "Сканировать Сцену", "Проверить URL", "СОХРАНИТЬ".
4. В программе нажмите **Пункт 3 (Восстановить файлы)** -- программа скопирует нужные файлы локально.

## ⚙️ Настройки (`renesance_config.txt`)

* **[sourceDirs]** — Папки с кэшем. Здесь программа ищет оригинальные картинки и 3д-модели. **ПРИОРИТЕТ ИМЕЕТ ЗНАЧЕНИЕ!** Папки проверяются сверху вниз. Положите папку с самой рабочей/актуальной версией игры в самый верх списка (Смотри секцию "Файлы-призраки").
* **[outputDir]** — Куда сливать результаты Полного экспорта. По умолчанию: `%APP_DIR%\Exported_Mods`.
* **[savedObjectsDir]** — Папка сохраненных объектов TTS (для Куба).
* **[savesDir]** — Папка для чтения JSON-карт.
* **[originalModsDir]** — Путь к родительской директории `Mods` (нужно ТОЛЬКО для работы Режима отладки).

Любые пути могут использовать стандартные переменные Windows (`%USERPROFILE%`).

## 🆘 Возможные проблемы и их решения

### При полном экспорте часть файлов "не нашлась"!
Если после экспорта в логах видно `[MISS] ... 1000 файлов не найдено`, на то есть две причины:
1. **Внешние ссылки.** Ссылки на Яндекс.Диск, Google Drive, нестандартные сайты и прочее. TTS не умеет кэшировать их как обычно, либо они вообще не качались физически вам на диск. Эти файлы восстановить локально нельзя.
2. **Файлы-призраки в кэше Steam.** Если у вас есть игра в Steam и Пиратка, кэш Steam мог сломаться, насоздавав пустых "нулевых" файлов-ссылок. Программа находит такой файл-призрак у вас в Steam-кэше, пытается скопировать его, а Windows выдает ошибку. 
   * **Решение:** Откройте `renesance_config.txt` и переместите путь к кэшу вашей *самой свежей Пиратской* игры в самый верх списка `[sourceDirs]`. Программа сначала заберет здоровые файлы оттуда, проигнорировав призраков!

### Не вставляется текст в меню программы
Кликните правой кнопкой мыши (ПКМ) в окно командной строки, текст должен вставиться.
Если все еще не работает - выберите Пункт в Настройках "Открыть конфиг в блокноте" и скопируйте пути туда напрямую.

## 🛠️ Сборка в EXE для публикации

Если вы хотите зашить приложение в один `.exe` файл, чтобы людям не нужен был установленный Node.js:
1. Выполните команду `npm install`
2. Выполните `npm run pack`
Готовый автономный файл появится в `dist/Renesance.exe`.
